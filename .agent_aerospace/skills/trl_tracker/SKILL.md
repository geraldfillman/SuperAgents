# TRL Tracker

Purpose: map test evidence and public disclosures into directional TRL movement.

Inputs:

- system name
- test event description
- milestone evidence
- supporting source URL

Outputs:

- `test_events` records
- `program_milestones` updates
- directional `trl_level` changes on `systems`

Priority signals:

- successful demonstration
- launch or flight test completion
- integration milestone
- certification or fielding milestone
